# Лабораторная работа №5 
**Тема:** Настройка GitHub Actions для автоматического запуска unit-тестов

---

## 1. Цель работы

Цель лабораторной работы — настроить механизм непрерывной интеграции (CI) с помощью GitHub Actions таким образом, чтобы после каждого `push` и при создании / обновлении pull-request автоматически запускались unit-тесты, написанные в предыдущей лабораторной работе.  
Дополнительно требуется обеспечить выполнение тестов на двух встроенных раннерах: **ubuntu-latest** и **windows-latest**.

---

## 2. Исходные данные

В качестве исходного проекта использовалась библиотека с функциями для работы с квадратом:

- файл `square.py` с функциями:
  - `area(a)` — вычисление площади квадрата;
  - `perimeter(a)` — вычисление периметра квадрата;
- файл с тестами `tests/tests_square.py`, использующий модуль `unittest` и проверяющий корректность работы функций.

Задача текущей лабораторной работы — добавить в репозиторий workflow-файл GitHub Actions и убедиться, что автотесты запускаются автоматически при изменении кода.

---
