# Лабораторная работа №4  
**Тема:** Написание unit-тестов (на примере модуля `square.py`)

## 1. Цели и задачи тестирования

Цель тестирования — проверить корректность работы функций модуля `square.py`, выполняющих вычисление площади и периметра квадрата.

Задачи тестирования:

- убедиться в правильности математических вычислений для различных входных данных (целые, вещественные, нулевые значения);
- проверить устойчивость функций к граничным значениям (сторона = 0);
- продемонстрировать использование встроенного фреймворка `unittest` для модульного тестирования в Python;
- сформировать отчёт по результатам тестирования.

## 2. Описание тестируемого продукта

Тестируемый продукт — модуль `square.py`, содержащий функции `area(a)` и `perimeter(a)`, выполняющие вычисления площади и периметра квадрата.

## 3. Область тестирования

Тестируются функции:
- `area(a)` при значениях 0, положительном целом числе, положительном вещественном числе;
- `perimeter(a)` при аналогичных значениях.

## 4. Стратегия тестирования

Используется модуль `unittest`.  
Запуск выполняется командой:

```
py -m unittest tests.tests_square
```

Проверка выполняется методами:
- `assertEqual`
- `assertAlmostEqual`

## 5. Критерии приёмки

Программа считается корректной, если:
- Все тесты выполняются успешно.
- Количество пройденных тестов совпадает с количеством сценариев.
- Функции соответствуют математическим формулам.
- Ошибки выполнения отсутствуют.

## 6. Ожидаемые результаты

Пример успешного вывода:

```
......
----------------------------------------------------------------------
Ran 6 tests in 0.000s

OK
```
